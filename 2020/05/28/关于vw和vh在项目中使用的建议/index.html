CSS3视口单位及自适应方案

2020-05-28 | technology
  • 各种视口单位介绍
  • 视口

    浏览器窗口的大小 不受显示器分辨率的影响

  • PX

    PX (pixel) 像素 是图像的基本采样单位

  • EM

    EM 是相对长度单位 始终都是相对父元素

  • REM

    REM 是相对长度单位 始终都是相对html根元素

  • VW

    VW 是可视窗口的宽度单位 1vw = 可视窗口的宽度的百分之一 即1-100

  • VH

    VH 是可视窗口的高度单位 1vh = 可视窗口的高度的百分之一 即1-100

  • VMIN

    VMIN 是选择vw和vh中最小的

  • VMAX

    VMAX 是选择vw和vh中最大的

VW VH 实现垂直居中

常用的自适应方案

  • 直接使用px JS改变meta 中的scale值 缺点是需要等页面加载完毕后 根据JS计算缩放比例再替换initial-scale的值 会多渲染一次页面 导致页面闪烁
  • 使用rem 根据写好的根元素来匹配对应元素的rem 缩放比例变动的时候 使用JS改变根元素的值 缺点是需要手动计算rem 而且除法会有小数缺省的问题，导致视觉上会有几个像素的偏差
  • 使用 vw 和 vh 的话不需要依赖JS进行计算执行 虽然在宽度和高度方向完全符合视口的自适应 但是很少会遇到高度是只有一屏的页面 因此Y轴方向的自适应意义不大 不过现在vw和vh有很好的兼容性了

整体建议

  • 从上面的介绍来看 vw和vh在全屏的页面兼容和自适应方面有高度的优势 既不用使用多余的JS计算 在兼容性方面也有保证 推荐整屏页面使用vw 和 vh 来代替rem
  • 在非全屏 和 需要滚动的页面来说 vw 在宽度的实现方面也比rem来的好 建议结合rem来设置高度 混合使用

其他

  • github上的postcss-px-to-viewport
  • 使用方法： 先npm install 然后在 .postcssrc.js 文件中添加配置
  • 跑dev 的时候发现所有的px都变成了vw和vh
  • 如果只需要单个文件使用该插件 按平常使用插件的方式引入就可以了
