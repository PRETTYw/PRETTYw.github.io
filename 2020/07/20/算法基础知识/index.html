算法的时间复杂度和空间复杂度

  • 时间复杂度：指执行当前算法所消耗的时间

    时间频度：一个算法花费的时间与算法中语句的执行次数成正比例，哪个算法中语句执行次数多，它花费时间就多。 一个算法中的语句执行次数称为语句频度或时间频度。记为 T(n)。
    算法中的基本操作语句的重复执行次数是问题规模 n 的某个函数，用 T(n)表示，若有某个辅助函数 f(n)，使得当 n 趋近于无穷大时，T(n) / f(n) 的极限值为不等于零的常数，则称 f(n)是 T(n)的同数量级函数。记作 T(n)= Ｏ( f(n) )，称Ｏ( f(n) ) 为算法的渐进时间复杂度
    即 O = T(n) / f(n)
    用常数 1 代替运行时间中的所有加法常数 T(n)=n²+7n+6 => T(n)=n²+7n+1
    修改后的运行次数函数中，只保留最高阶项 T(n)=n²+7n+1 => T(n) = n²
    去除最高阶项的系数 T(n) = n² => T(n) = n² => O(n²)
    常见的算法时间复杂度由小到大依次为：Ο(1)＜Ο(log2n)＜Ο(n)＜Ο(nlog2n)＜Ο(n^2)＜Ο(n^3)＜Ο(n^k)＜Ο(2^n)

常数阶 O(1)

  • 无论代码执行了多少行，只要是没有循环等复杂结构，那这个代码的时间复杂度就都是O(1)

对数阶 O(log2n)

  • 在while循环里面 假设循环x次之后，i 就大于 n 了，此时这个循环就退出了，也就是说 2 的 x 次方等于 n，那么 x = log2n也就是说当循环 log2n 次以后，这个代码就结束了。因此这个代码的时间复杂度为：O(log2n)

线性阶 O(n)

  • for循环里面的代码会执行n遍，因此它消耗的时间是随着n的变化而变化的，因此这类代码都可以用O(n)来表示它的时间复杂度。

线性对数阶 O(nlogN)

  • for循环里面嵌套while循环 将时间复杂度为O(logn)的代码循环N遍的话，那么它的时间复杂度就是 n * O(logN)，也就是了O(nlogN)。

平方阶 O(n²)

  • for循环里面的代码会执行n遍，把 O(n) 的代码再嵌套循环一遍，它的时间复杂度就是 O(n²) 即 O(n²) 如果将其中一层循环的n改成m，那它的时间复杂度就变成了 O(m*n)

  • 空间复杂度：指执行当前算法需要占用多少内存空间

    从用户使用体验上看，更看重的程序执行的速度。一些缓存产品(redis, memcache)和算法(基数排序)本质就是用空间换时间
    在做算法分析时，主要讨论的是时间复杂度
    计算公式记作：S(n)=O(f(n))，其中，n为问题的规模，f(n)为语句关于n所占存储空间的函数。

什么是哈希表 Hash

  • 是一种根据关键字直接访问内存存储位置的数据结构 通过哈希表，数据元素的存放位置和数据元素的关键字之间建立起某种对应关系，建立这种对应关系的函数称为哈希函数
  • key值不可以重复，value可以重复
  • 对象具有天然的哈希特性

什么是链表

  • 链表是一组节点组成的集合，每个节点都使用一个对象的引用来指向它的后一个节点。指向另一节点的引用讲做链
  • 单向链表：用一组任意的内存空间去存储数据元素 每个节点(node)都由数据本身和一个指向后续节点的指针组成 整个链表的存取必须从头指针开始，头指针指向第一个节点 最后一个节点的指针指向空（NULL）
  • [head][next] —> [数据][next] —> [数据][next] —> [数据][next] —> [null]
  • 双向链表：previous属性，让其指向前驱节点的链接
  • [head][next][prev] — [数据][next][prev] — [数据][next][prev] — [null]
  • 循环链表：循环链表和单链表相似，节点类型都是一样，唯一的区别是，在创建循环链表的时候，让其头节点的 next 属性执行它本身
  • [head][next] — [数据][next] — [数据][next] — [head][next]

什么是栈 （stack）

  • 栈是一种后进先出的数据结构 一种遵循后进先出原则的有序集合 也就是说最新添加的项最早被移出；它是一种运算受限的线性表，只能在表头/栈顶进行插入和删除操作
  • 栈有栈底和栈顶
  • 向一个栈插入新元素叫入栈（进栈），就是把新元素放入到栈顶的上面，成为新的栈顶
  • 从一个栈删除元素叫出栈，就是把栈顶的元素删除掉，相邻的成为新栈顶
  • JS也为数组提供了方法可以实现类似入栈出栈功能 入栈push()、 出栈pop()
  • 又名堆栈或堆叠 是堆的一部分 限定仅在表尾进行插入和删除操作的线性表
  • 由操作系统自动分配释放 ，存放函数的参数值，局部变量的值等。其操作方式类似于数据结构中的栈。

什么是堆 （heap）

  • 是计算机科学中一类特殊的树状数据结构
  • 堆总是一棵完全二叉树 某个节点的值总是不大于或不小于其父节点的值
  • 根节点最大的堆叫做最大堆或大根堆，根节点最小的堆叫做最小堆或小根堆
  • 堆是在程序运行时，而不是在程序编译时，申请某个大小的内存空间
  • 堆是指程序运行时申请的动态内存
  • 一般由程序员分配释放， 若程序员不释放，程序结束时可能由OS回收，分配方式倒是类似于链表。

什么是队列

  • 队列是一种先进先出的数据结构。 队列在列表的末端增加项，在首端移除项
  • 在表的首端（队列头）进行删除操作，在表的末端（队列尾）进行插入操作
  • JS为数组提供了方法可以实现队列功能， 入队unshift()、 出队pop()

什么是正则

  • 英文（regular expression）表达规范和规则的句子
  • JS中的RegExp构造函数 是正则两个单词的前三个字母缩写
  • JS中正则有两个构造方法：test 和 exec 分别是是否符合和哪里符合
  • 元字符：描述字符的字符，比如，数字，字母，空格，换行等
  • 元字符大多以反斜杠开头 \ 正则表达式字面量写法是用两个斜杠包裹 \ sth\
  • \d，匹配单个数字 digit / 数字 缩写
  • \w，匹配单个单词字符 word / 单词 缩写 字母 a-z，数字 0-9
  • \s，匹配单个空白字符 space / 空白 缩写
  • \n，匹配换行符 newline / 换行 缩写
  • \r，匹配回车符 return / 回车 缩写
  • \t，匹配制表符 table / 表格 缩写
  • \b，匹配单词边界 boundary / 边界 缩写
  • 上面只能表示单字符 所以有了量词
  • n+，至少1个 n 类型的字符
  • n*，0到多个 n 类型的字符
  • n?，0 或 1 个 n 类型的字符
  • n{X}，X 个 n 类型的字符
  • n{X,Y}，X 到 Y 个 n 类型的字符
  • n{X,}，至少 X 个 n 类型的字符
  • n$，以 n 类型的字符结尾
  • ^n，以 n 类型的字符开头
  • 再来就是一些规定的常用的表达式
  • [abc]，匹配单个字符，abc 的元素集合
  • [^abc]，匹配单个字符，不是abc 的元素集合
  • [0-9]，匹配单个字符，从0到9 的元素集合
  • [a-z]，匹配单个字符，26 字母 的元素集合

什么是动态规划

  • dynamic programming
  • 是运筹学的一个分支
  • 是解决多阶段决策过程最优化的一种数学方法
  • 把多阶段问题变换为一系列相互联系的的单阶段问题，然后逐个加以解决
  • 即将一个问题拆成几个子问题，分别求解这些子问题，即可推断出大问题的解

什么是位运算

  • 位运算符是在数字底层（即表示数字的 32 个数位）进行操作的。
  • 整数有两种类型，即有符号整数（允许用正数和负数）和无符号整数（只允许用正数）
  • 有符号整数使用 31 位表示整数的数值，用第 32 位表示整数的符号，0 表示正数，1 表示负数。
  • 前 31 位中的每一位都表示 2 的幂，从第 1 位（位 0）开始，表示 20，第 2 位（位 1）表示 21。没用到的位用 0 填充，即忽略不计
  • 负数也存储为二进制代码，不过采用的形式是二进制补码：确定该数字的非负版本的二进制表示 / 求得二进制反码，即要把 0 替换为 1，把 1 替换为 0 / 在二进制反码上加 1

    位运算NOT

  • 由否定号（~）表示 位运算 NOT 实质上是对数字求负，然后减 1

    位运算 AND

  • 由和号（&）表示 把每个数字中的数位对齐 然后1 1 返回1 / 1 0 返回0 / 0 0 返回0

    位运算 OR

  • 由符号（|）表示 把每个数字中的数位对齐 然后1 1 返回1 / 1 0 返回1 / 0 0 返回0

    位运算 XOR

  • 由符号（^）表示 把每个数字中的数位对齐 然后1 1 返回0 / 1 0 返回1 / 0 0 返回0

    左移运算

  • 由两个小于号表示（<<）它把数字中的所有数位向左移动指定的数量

    右移运算

  • 由两个大于号表示（>>）它把 32 位数字中的所有数位整体右移，同时保留该数的符号（正号或负号）

什么是树

  • 线性结构一般是一对一关系 一对多关系就是特殊的数据结构 — 数 做的事情
  • 是n个结点的有限集 n=0时称为空树
  • 有且仅有一个称为根的结点
  • 树的结点包含一个数据元素及若干指向其子树的分支 结点拥有的子树数称为结点的度 度为0的结点称为叶结点或终端结点 度不为0的结点称为非终端结点或分支结点
  • 如果将树中结点的各个子树看成从左至右有次序的 不能互换的 称为有序树
  • 森林就是m个互不相交的树的集合 对树中的每个结点而言 其子树的集合就是森林

什么是二叉树

  • 树的一种 每个节点最多只能有两个子节点的树叫做二叉树
  • 子节点分为左节点和右节点
  • 如果二叉树的所有叶子节点都在最后一层 并且结点总数为2^n-1 n为层数 则称该树为满二叉树
  • 如果二叉树的所有叶子节点都在最后几层 并且叶子节点连续 称为完全二叉树
  • 遍历二叉树有前序 中序和后序三种遍历方法 属于DFS
  • 前序：先输出父节点 再遍历左子树和右子树
  • 中序：先遍历左子树 再输出父节点 再遍历右子树
  • 后序：先遍历左子树 再遍历右子树 最后输出父节点
  • 遍历二叉树有层次遍历方法 属于 BFS

什么是DFS Depth-First-Search

  • 深度优先遍历（DFS）也叫深度优先搜索
  • 不断地沿着顶点的深度方向遍历 顶点的深度方向指的是邻接点方向
  • 用于查找从根结点到目标结点的路径
  • 在 DFS 中找到的第一条路径并不总是最短的路径
  • DFS 都可以使用栈来简化操作，并且其实树本身是一种递归的数据结构

什么是BFS Breadth-First-Search

  • 宽度优先遍历（BFS）也叫广度优先搜索
  • 并不考虑结果的可能位置，彻底地搜索整张图，直到找到结果为止。
  • 属于一种盲目搜寻法，目的是系统地展开并检查图中的所有节点，以找寻结果
  • BFS搜索过程中遇到的解一定是离根最近的 一定就是最优解
