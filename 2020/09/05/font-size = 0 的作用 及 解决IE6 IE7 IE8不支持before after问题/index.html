font-size = 0 的作用 及 解决IE6 IE7 IE8不支持before after问题

2020-09-05 | technology
  • 空白节点莫名其妙的两个非块级元素之间，总会出现那么几个像素的空白
  • 问题的根源是 inline（a标签默认是display:inline） 和 inline-block (.list 设置的是 display:inline-block) 是内联布局，既然是内联那么就会受空白区域的影响。
  • 元素节点有文本节点，在缩进代码时会占据宽度
  • 在实际开发中，为了更好的还原设计稿，在父元素很有必要设置font-size:0，避免莫名其妙的间距。
  • 也可以去掉标签的空白部分来解决
