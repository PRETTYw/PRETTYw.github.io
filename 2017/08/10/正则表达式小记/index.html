正则表达式小记

2017-08-10 | technology

平时会看到很多匹配规则，但是没有仔细去研究，有时候写html有些表格内容需要定制，总去网上找确实麻烦，然后就在慕课网学了篇关于正则表达式的课程，下面记录一下正则表达式的基本规则，防止忘记~

开篇

使用正则表达式可以在IDE中处理规则复杂的字符串查找、替换需求 （比如 .find./-name*.txt 就是在当前目录下查找所有的TXT文件），也可以在JavaScript程序设计中使用正则表达式处理字符串，以及定制一些自己想要的匹配需求。

正则表达式英文名称：Regular Expression ，主要是使用单个字符串来描述、匹配一系列符合某个句法规则的字符串（按照某种规则去匹配符合条件的字符串）

JavaScript通过内置对象regexp支持正则表达式，有两种方式实例化regexp对象：字面量和构造函数

第一种
var reg=/\bis\b/;
‘He is a boy. This is a dog.Where is she?’.replace(reg,’IS’)
输出结果是：”He IS a boy.This is a dog.Wherre is she?”
var reg=/\bis\b/g;
‘He is a boy. This is a dog.Where is she?’.replace(reg,’IS’)
输出结果是：”He IS a boy.This IS a dog.Wherre IS she?”

第二种
var reg=new RegExp(‘\bis\b’);
‘He is a boy. This is a dog.Where is she?’.replace(reg,’IS’)
输出结果是：”He IS a boy.This is a dog.Wherre is she?”
var reg=new RegExp(‘\bis\b’ ‘g’);
‘He is a boy. This is a dog.Where is she?’.replace(reg,’IS’)
输出结果是：”He IS a boy.This IS a dog.Wherre IS she?”**

第三种
‘He is a boy.Is he?’.replace(/\bis\b/g,’0’)
输出结果是：”He 0 a boy.0 he?”

正则表达式的详细介绍

正则表达式有三个修饰符：g（global） 表示全文搜索，如果不添加的话，默认匹配完第一个就会停止；i（ignore case）表示忽略大小写，默认大小写是敏感的；m（multiple lines） 表示多行搜索，默认匹配当前行。

对象属性除了上面三个外还有lastIndex：是当前表达式匹配内容的最后一个字符的下一个位置；source：正则表达式的文本字符串

正则表达式由两种基本字符类型组成：原义文本字符和元字符； 元字符是在正则表达式中有特殊含义的非字母字符

\t 表示水平制表符
\t 表示垂直制表符
\n 表示换行符
\r 表示回车符
\o 表示空字符
\f 表示换页符
\cX 表示与X对应的控制字符（ctrl+X）

我们使用元字符[]来构建一个简单的类，而所谓的类是指符合某些特性的对象，是泛指，不是特指某个字符；（表达式[abc]把字符a或b或c归为一类，表达式可以匹配到这类的字符）；

字符串取反加上^（表达式[^abc]表示不是字符a或b或c的内容）；

正则表达式提供了范围类：我们使用 [a-zA-Z]表示包括大小写的字母匹配，是个闭区间；

正则表达式提供了预定义类来匹配常见的字符:

( 字符. 等价类为 [^\r\n] 含义是除了回车符和换行符之外的所有字符 )
( 字符\d 等价类为 [0-9] 含义是数字字符 )
( 字符\D 等价类为 [^0-9] 含义是非数字字符 )
( 字符\s 等价类为 [\t\n\xOB\f\r] 含义是空白字符 )
( 字符\S 等价类为 [^\t\n\xOB\f\r] 含义是非空白字符字符 )
( 字符\w 等价类为 [a-zA-Z_0-9] 含义是单词字符 )
( 字符\W 等价类为 [^a-zA-Z_0-9] 含义是非单词字符 )
从上面也可以看出正则的大小写很重要，同时，正则表达式还提供了几个常用的边界匹配字符
字符^表示以XXX开始
字符$表示以XXX结束
字符\b表示单词边界
字符\B表示非单词边界
字符？表示出现零次或一次（最多出现一次）
字符+表示出现一次或多次（至少出现一次）
字符*表示出现零次或多次（任意次）
字符{n}表示出现n次
字符{n,m}表示出现n到m次
字符{n,}表示至少出现n次

正则表达式有个贪婪模式，就是如果你想匹配一个数字出现3-6次，那在匹配的时候，它一定是按6次匹配的，就是想要尽可能的多匹配：

‘12345678’.replace(/\d{3,6}/g,’X’)
结果是”X78”

但是有时候我们不想要贪婪模式的匹配结果，为了让正则表达式尽可能少的匹配，也就是说一旦成功匹配后就不再继续尝试，这就是非贪婪模式，做法是只要在量词后面加上?即可：

‘123456789’.match(/\d{3,5}?/g)
结果是[“123”,”456”,”789”]

如果我们想匹配一个字符串wang连续出现三次的情况，正则表达式也提供了量词供我们使用，(wang){3},但是如果不加()，量词只会关心紧挨着它的单词，也就是匹配3次g，所以记得加上具有分组功能的(),使用|可以达到或的效果，我们可以使用$123来当作变量，匹配某些反向引用的问题：

假如我们想把 2017-08-11 匹配换成 08/11/2017
‘2017-08-11’.replace(/(\d{4}-(\d{2})-(\d{2})/g,’$2/$3/$1’)
如果我们不希望捕获某些分组，只需要在分组内加上?:就可以了(?:wang)

正则表达式从文本头部向尾部开始解析，文本尾部方向称为”前”，前瞻就是在正则表达式匹配到规则的时候，向前检查是否符合断言，后顾/后瞻方向相反，JavaScript不支持后顾，符合和不符合特定断言称为肯定/正向匹配和否定/负向匹配

正则表达式有个方法：
1：RegExp.prototype.test(str) 用于测试字符串参数中是否存在匹配正则表达式模式的字符串，如果存在返回true，否则返回false
2：RegExp.prototype.exec(str) 使用正则表达式模式对字符串执行搜索，并将更新全局RegExp对象的属性以反映匹配结果，如果没有匹配的文本则返回null，否则返回一个结果数组：index声明匹配文本的第一个字符的位置，input存放被检索的字符串string；
3：String.prototype.search(reg) search()方法用于检索字符串中指定的子字符串，或检索与正则表达式相匹配的子字符串，方法返回第一个匹配结果index，查找不到返回-1，search()方法不执行全局匹配，它忽略g标志，并且总是从字符串的开始开始检索；
4：String.prototype.match(reg) match()方法将检索字符串，，以找到一个或多个与regexp匹配的文本，regexp是否具有标志g对结果影响很大；
5：String.prototype.split(reg) 使用split方法把字符串分割为字符数组，而在一些复杂的情况下我们可以使用正则表达式解决；
6：String.prototype.replace 方法有三种传参方式

String.prototype.replace(str,replaceStr)
String.prototype.replace(reg,replaceStr)
String.prototype.replace(reg,function)
function会在每次匹配替换的时候调用，传四个参数，分别是：匹配的字符串、正则表达式分组内容，没有分组则没有该参数、匹配项在字符串中的index和原字符串；
