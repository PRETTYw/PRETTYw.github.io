最近一直在积极准备面试，然后在慕课网上购买了一个前端面试技巧的专题，看完之后很有感触，所以打算写个文章整理一下自己在视频上面学到的，顺便加上后来自己找的一些面试题资料，这样方便以后可以系统的拿出来复习~

对于jd描述，社招一定要看并且分析，对于校招可以忽略，简历可以对照jd改写出相吻合的简历，对于未掌握的技术栈快速复习、理解，自我介绍一定要打草稿，展示什么优势、描述什么项目，切忌临场发挥。对于一面来说，主要是重基础、懂原理、要思考、知进退、这样一定能势不可挡，对于二面来说，要懂得横向扩展、项目结合，做到有的放失，对于三面来说，需要有经验、懂合作、有担当、懂规矩，会察言观色，对于终面来说，要会沟通，要上进、好性格、有主见、强逻辑，这样的才无可挑剔，最后要胜不骄、败不馁、总结经验、步步为营，争取多拿offer~

谈兴趣：一定要跟前端沾边，围绕前端技术来谈
难题：我要回去思考一下这个问题，能不能指点一下，需要参考什么资料，我想把这个弄懂，对于太难的面试，就抱着需要有收获的心态去请教，讨论要什么技术资料，假如面试失败只是暂时的，但是把知识体系补全了也赚了

看下面之前得把自己以前学的笔记什么的都看一遍的为好，很多基础知识在这里就不详细说了，主要说些面试题，以及推荐一些复习网址

一面/二面

这里的一面和二面一般包括基本面试和笔试

面试的时候要注意：准备要充分，知识要系统，沟通要简洁、内心要诚实，态度要谦虚，回答要灵活

复习知识按：页面布局，css盒模型，dom事件，http协议，面向对象，原型链，通信，安全，算法来准备

二面/三面

这里的二面和三面一般包括基本笔试和深度笔试

面试的时候要注意：知识面要广，理解要深刻，内心要诚实，态度要谦虚，回答要灵活，要学会赞美

复习知识除了上面的，还包括：渲染机制，js运行机制，页面性能，错误监控

三面/四面

这里的三面和四面一般包括深度笔试和技术总监面试

面试的时候要注意：准备要充分，描述要演练，引导找时机，优势要发挥，回答要灵活

复习知识除了上面的，还包括：业务能力，团队协作能力，事务推动能力，带人能力，其他能力，比如学习、组织、行业经验等

终面

这里的终面一般是boss直面或者拥有一票否决权的主管面试

面试的时候要注意：乐观积极，主动沟通，逻辑顺畅，上进有责任心，有主张，做事果断

需要复习的知识就是：职业竞争力和职业规划

什么是闭包

变量作用域：全局变量（函数内部的可以读）和局部变量（外无法读取内部），不用var声明的是全局变量，为了获取函数内部的局部变量，闭包就是能够读取其他函数内部变量的函数（内外沟通的中介）

display属性

规定元素应该生成的框的类型，none block inline

什么是原型和原型链

原型+prototype+方法 ，原型对象仅用一份，用途是为每个实例对象存储共享的方法和属性， 原型链就是实例对象的proto属性

页面布局

总的来说，web布局经历了以下几种阶段：
1、table布局：通过dw拖拽或手写table
和表格相关的属性：table-cell和table-row
display: table;border-collapse: collapse;声明table布局同时为表格设置合并边框模型，在子元素中通过设置display: table-cell来进行布局。

2、div、span、position、float布局：借助元素盒模型及浮动定位属性进行布局
3、双飞翼和圣杯布局
4、flex弹性布局：解决排列方向、对齐方式、自适应尺寸
5、bootstrap中的栅格布局：提供响应式解决方案

css盒模型

页面就是由一个个盒模型堆砌起来的，每个HTML元素都可以叫做盒模型，盒模型由外而内包括：边距（margin）、边框（border）、填充（padding）、内容（content）。它在页面中所占的实际宽度是margin + border + paddint + content 的宽度相加。

不同之处就是标准盒模型的内容大小就是content的大小，而IE的则是content + padding +border 总的大小。通过设置 box-sizing:content-box(W3C)/border-box(IE)就可以达到自由切换的效果。

边距重叠：两个box如果都设置了边距，那么在垂直方向上，两个box的边距会发生重叠，以绝对值大的那个为最终结果显示在页面上。

BFC的基本概念–BFC就是“块级格式化上下文”的意思，也有译作“块级格式化范围”。它是 W3C CSS 2.1 规范中的一个概念，它决定了元素如何对其内容进行定位，以及与其他元素的关系和相互作用。通俗的讲，就是一个特殊的块，内部有自己的布局方式，不受外边元素的影响。

BFC原理–

BFC内部的盒子，会在垂直方向，一个接一个地放置。垂直方向上也会发生边距重叠。
BFC就是页面上的一个独立容器，容器里面的子元素不会影响到外面的元素，外边的也不会影响里边的。
BFC的区域不会与float box重叠。
计算BFC的高度时，浮动元素也被计算在内。

BFC如何产生–

  • overflow: auto/ hidden;
  • position: absolute/ fixed;
  • float: left/ right;
  • display: inline-block/ table-cell/ table-caption/ flex/ inline-flex
    在属性值为这些的情况下，都会让所属的box产生BFC。
