angular环境配置

2018-04-10 | technology

如果电脑是以前就有安装过node的话，建议先卸载，控制面板\所有控制面板项\程序和功能里面找到nodejs进行卸载，当然环境也要删除干净，比如注册表什么的，可以下载一个reg clean进行管理。

安装新node

去node官网下载安装包，最好下载的是稳定版的node版本，点击安装程序后按照指引一步步来就好了，新版的node是默认安装好node和npm的，都安装好后再cmd中看看版本号是不是对的就好了，如果都安装好了就没有问题，版本号都会给出来的，在cmd中输入输入”node“进入node开发模式下输入：console.log(‘hello world’)，会打印出hello world说明node已安装成功。

安装cnpm、cli####

因为cli里面有一下资源需要翻墙下载，所以我们可以安装使用淘宝镜像来解决问题：在cmd中输入 npm install -g cnpm –registry=https://registry.npm.taobao.org 进行安装，安装成功后就需要安装cli，如果以前安装过可以用 npm uninstall -g @angular/cli 来卸载以前安装的cli，然后使用 npm cache clean 清除一下缓存，接下来使用 cnpm install -g @angular/cli@latest 镜像安装cli，安装完成后使用ng v 查看是否安装成功。

新建项目

使用命令行ng new projectName 就可以创建一个新的angular项目，当然你想在哪个目录下创建就要把cmd cd到相应的目录下哟。过程会下载很多东西，耐心等待就好了，当成功创建完成后，使用ng serve命令就可以启动项目，默认运行在4200端口，要是想改的话也是可以的，打开浏览器，输入localhost:4200就可以访问到，如果成功的话说明你的angular环境已经配置好，如果不行的话，还需要找找原因，重新装多几次。

过程中可能会出现很多错误，比如报错什么的，一定要记住你的项目根目录是什么，只要认定这个点，它报什么错，就照着解决就好了。项目和node放在不一样的位置没关系，记清楚全局安装到哪里，项目安装到哪里就好了。

错误问题

1、’sudo’不是内部或外部命令，也不是可运行的程序或批处理文件
第一步：在任意目录中新建文本文件，命名为sudo.js
第二步：用记事本打开sudo.js文件，粘贴下方的代码。
var command = WScript.Arguments.Item(0);
var argument = “”;
for (var i = 0; i < WScript.Arguments.Count(); ++i){ argument += WScript.Arguments.Item(i) + “ “; } try{ var shellapp = new ActiveXObject(“Shell.Application”); shellapp.ShellExecute(command, argument, null, “runas”, 1); } catch(e){ WScript.Echo(“Something wrong: “ + e.description + “ By http://www.alexblair.org"); }
现在当需要在命令行里提权操作时，就非常简单了。只要在该文件上单击右键，选择“在命令提示符中打开”即可。
然后执行 sudo命令就可以正常执行了。

sudo是linux命令，windows中应直接省略。否则回报sudo不是内部命令
